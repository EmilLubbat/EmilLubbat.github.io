Covid Cases

This project explores the relationtionship between
the rate of covid cases and the death rate caused by covid
in countries around the world.

Data collection and cleaning

The data used in this project has been obtained from pulicly acessable data on Kaggel.com. Although this data has already been compiled, it is neccessary to clean the data so as to only use that which will be analysed and maintain good data standards.
In this example, there are many fields which are irellevant to the analysis and can be easily cleansed. There are also records which overlap with others when grouped. These records however are only in a few cases which can be easily amended with an extra line of code.

When importing any data it is always useful to check that the data is stored in the appropriate format. This can be done within the database if the data is stored in a spreadsheet; while importing into a SQL software using the programs tools; or by casting within a query.

Data Mapping

Microsoft Power BI and Tableau Desktop can be connected directly to the SQL server, automatically importing all of the tables created and will update with live changes to the data or queries. Tableau Public, however, cannot connect directly to SSMS so each table needs to be imported as CSV files.

After importing these tables into Tableau or Power BI, the relationships between the tables needs to be identified using a data model. In this case: the HighestInfections and HighestDailyInfections tables share the Location fields; the HighestDailyInfections and TotalDeathPerContinent tables share the Continent fields; the TotalDeathPerContinent and GlobalDeathPerCase tables share the total_cases fields.
In this case, the data model will also help to make the dashboard interactive.

The dashboard will show the relationship between covid cases and deaths for different countries around the world so only the queried data relating to infection, deaths and location highlighted in the data model above will be used to build the dashboard.

Now the dashboard can be built by adding visualisations of the data.

Data Visualisations

Firstly the GlobalDeathPerCase table can be simply represented with their values in cards or a table as they are just total values with no other variables.
The TotalDeathPerContinent table can also be displayed as a simple bar chart as it is measuring the quantitative variable of total deaths, which is on the y-axis, against the qualitative vairiable of Continent, which is on the x-axis.
Next, the HighestInfections table can be shown with a map, using the PercentPopulationInfection field as the legend to display the severity of infcetion of each county around the world.
Lastly, the HighestDailyInfections table can be plotted into a line chart as both the measured variables of time, on the x-axis, and infections, on the y-axis, are continuous variables.


To show seperate lines for different countries, the location field should be added to the legend. Displaying every country on this chart would make the visualisation unreadable so the location must be filtered to only display the top 10 countries with the highest PercentPopulationInfection in this case.

After arranging these visualisations on the dashboard, adding navigation buttons and setting some visualisations to be used as filters will make the dashboard interactive.
This is possible due to the relationships between the charts that were assigned in the data model before.

Here, the buttons for the UK and Spain have been selected so the data from these countries can be easily compared.
